## Combining New 3R Models

The discussion confirmed that combining strengths of new 3R models is important, but must be separated into two levels.

### 3R Composer L1: System Composer

Purpose:

```text
Different inputs / failure modes -> choose the right 3R model -> unify outputs and evidence reports.
```

Representative strengths:

- DUSt3R / MASt3R: pair images, matching, static geometry.
- Fast3R: many-view one-forward scaling.
- Spann3R: spatial memory / global pointmap.
- CUT3R / STream3R / LONG3R: streaming and long-context.
- MonST3R / POMATO / D^2USt3R: dynamic and 4D.
- Test3R / TTT3R: test-time consistency and adaptation.
- Splatt3R / NoPoSplat / InstantSplat: teacher-visible Gaussian / NVS demo.

Value:

- near-term KYKT support
- capability cards
- model routing
- evidence reports
- benchmark organization

Risk:

- may be engineering-only if it does not produce a new mechanism.

### 3R Composer L2: Mechanism Distillation

Purpose:

```text
Extract mechanisms from multiple models and convert them into unified 3R spatial-intelligence actions.
```

Examples:

- Spann3R / Point3R -> external spatial memory.
- CUT3R / STream3R -> persistent streaming state.
- LoGeR / Mem3R -> hybrid memory.
- PAS3R / FILT3R -> pose-adaptive / Kalman-style update.
- OVGGT / LongStream -> cache and anchor protection.
- Test3R / TTT3R -> geometry consistency and test-time revision.
- MonST3R / POMATO / D^2USt3R -> dynamic/static split.
- Splatt3R / NoPoSplat -> pose-free visual asset surface.

Unified actions:

- write memory
- protect anchor
- evict cache
- run critic
- revise hypothesis
- split dynamic/static
- reroute model
- request global context
- generate visual asset

This L2 layer is the stronger path toward paper-grade novelty.
