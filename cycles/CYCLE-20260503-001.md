## Form Considered

Three forms were considered before writing:

A. Single new `paradigm/RESEARCH_CODE_DISCIPLINE.md`, wired into the load protocol.

B. Inline edits to `RESEARCH_PARADIGM.md` and `RESEARCH_SKILL_RULES_DRAFT.md`.

C. Claude Code skill at `.claude/skills/dream-discipline/SKILL.md`.

Selected: **A**.

Reasoning:

- A keeps Dream's existing topical structure (`paradigm/` already houses paradigm, workflow, data model, skill rules draft, content roadmap). One more file with a distinct role fits cleanly.
- B would push two already-long files (`RESEARCH_PARADIGM.md` 314 lines, `RESEARCH_SKILL_RULES_DRAFT.md` 234 lines) past the point where their focus stays clear, and would dilute the Karpathy provenance.
- C is premature. `RESEARCH_SKILL_RULES_DRAFT.md` is explicitly drafted with a future-skill packaging step gated by user approval; doing C now would jump that gate. A standalone discipline file can later be folded into the skill package once stable.

The user did not push back on form A in the same turn after the form was previewed.

## Adaptation Decisions

The discipline file maps Karpathy's four principles to Dream context:

| Karpathy principle | Dream rule | Why adapted |
|---|---|---|
| Think Before Coding | Think Before Synthesizing | research synthesis is more vulnerable to silent assumptions because most evidence is `inferred` or `speculative`; promoted evidence-label discipline from a schema field to default behavior |
| Simplicity First | Minimum Viable Mechanism | combats the 3R-specific risk of stacking 4-5 fashionable mechanisms; tied to the existing A1-A8 / P1-P8 vocabulary so simplicity is measurable |
| Surgical Changes | Surgical Edits | Dream has a thick cross-link graph; "while I'm here" edits are the highest-blast-radius failure mode; codified the trace-test for every changed line |
| Goal-Driven Execution | Falsifiable Research Goals | promoted Dream's existing `success_criteria` and `stop_conditions` schema fields to non-skippable; required cycle-log header fields (failure mode, owned actions, proxy metric, fail-fast condition) |
| (Karpathy hint via "Don't hide confusion") | Honesty Override (Dream-native fifth rule) | research artifacts have a stronger truth-tracking requirement than typical product code; explicit override that beats neat narrative |
