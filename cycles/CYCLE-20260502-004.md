## Rationale

The previous cycle (`CYCLE-20260502-003`) produced `planning/BRANCH_SHORTLIST_DECISION_SURFACE.md` and updated the agent handoff prompt to begin at the shortlist stage. The next operationally cheap step is template preparation, because:

- `planning/ACTION_TAXONOMY_AND_PROXY_METRICS.md` defines A1-A8 and P1-P8 abstractly but does not yet provide a per-case schema
- different branches (Geometry Critic, Executive Memory, Composer, Dynamic Object Permanence, Cross-Modal, Active Perception) need to share a single case card schema for cross-branch comparison
- mechanism specs must be written against a stable form so that comparison and approval gates remain consistent

The two new templates are branch-neutral. They do not select, rank, or discard any branch.

## Proxy Case Card Template Summary

`templates/proxy_case_card.md` records, per scenario:

- proxy id (P1-P8)
- linked failure modes (F1-F6)
- linked actions (A1-A8)
- linked candidate branch(es)
- input artifact source and evidence label
- evidence signal vector values
- comparator policies and the Dream policy with predicted proxy values
- predicted proxy outcome and useful-signal threshold
- writing value (which related-work / figure / novelty claim it supports)
- explicit boundaries (no reproduction, no training, no frontend change)
- next action

## Finalist Mechanism Spec Template Summary

`templates/finalist_mechanism_spec.md` records, per user-approved branch:

- approval status and decision id
- failure modes addressed
- owned and support actions
- comparator anchors and weakest comparator pressure
- core claim and one-line thesis
- mechanism pseudocode (inputs, state, triggers, action logic, outputs)
- action policy table (when each owned action fires)
- proxy validation plan (primary + secondary proxy, case cards, thresholds)
- teacher demo form
- KYKT integration surfaces and frontend handoff requirement
- engineering cost across L1-L4 (L3/L4 marked blocked until approval)
- risks and evidence labels
- explicit boundaries
- linked artifacts and next step
