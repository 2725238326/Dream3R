### Root cause of 2-window zero

The sliding branch requires sufficient bank occupancy to produce a meaningful local-window context. At window 0–1, bank has only ~40 entries (below the sliding window span); the gate correctly routes 0 weight to sliding. Once bank saturates (K=256, achieved by window 12), the sliding branch consistently receives ~28% weight.

**This is correct routing behavior, not a bug or training-free bias.**
