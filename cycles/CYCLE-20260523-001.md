Interpretation:

- The ablation confirms the A6 conclusion across both recurrence backends: sliding is used once enough long-context evidence is available.
- The 10-window Mamba NSA case is the only NSA-positive zero-sliding case; the 30-window and 52-window Mamba cases both fire, so this is a short-horizon / routing-threshold effect rather than a branch wiring failure.

### Experiment 2: 128-window long-sequence (kitti_200win_a6_evidence.json)

- Config: mamba_hybrid, NSA enabled, max_sequences=4, 128 windows realized
- **Sliding branch fire rate: 128/128 = 100%**
- Branch weight distribution: compressed=0.035, selected=0.685, **sliding=0.280**
- Bank occupancy: 40 (win 0) → 256 (win 12+, capacity saturated)
- Effective top-k: avg=11.4 (range 8–13)
- Drift range: [-0.077, 0.588] — stable, no divergence at 128 windows
- Per-window latency: avg 2.1s

### Root cause of 2-window zero

The sliding branch requires sufficient bank occupancy to produce a meaningful local-window context. At window 0–1, bank has only ~40 entries (below the sliding window span); the gate correctly routes 0 weight to sliding. Once bank saturates (K=256, achieved by window 12), the sliding branch consistently receives ~28% weight.

**This is correct routing behavior, not a bug or training-free bias.**
