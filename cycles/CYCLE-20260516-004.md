## Observation: G4 negation-context grep tension

Cycle 039 surfaced a tension already noted in cycle 036 but now exercised at scale: candidate-not-final 句式 pedagogy ("不说 X, 说 Y") explicitly uses the forbidden patterns to teach readers what NOT to say. The G4 grep is too coarse to distinguish "forbidden over-claim" from "教学-用法 forbidden pattern in negation context".

Two paths forward (decision deferred to cycle 040 / 041 if pattern recurs):

- **Path A (continue per-cycle corrective rephrasing)**: each new section drafting rephrases negation-context forbidden patterns to remove literal regex substrings (e.g., "最终架构方案" → "最终方案"; "X 解决了" → "X 已完全解决"). Pro: G4 stays binary strict; advisor reading still recognizes 候选-not-final intent. Con: rephrasings (e.g., "宣称 ... 优于") may read awkward in academic Chinese.
- **Path B (document G4 negation-context exemption in STYLE_CONTRACT §5)**: extend §5 句式对比表 with an explicit note that 候选-not-final 句式 examples ("不说 X 解决了 ..., 说 X 提供了 ...") may use forbidden patterns inside quoted-as-bad-example contexts; G4 grep then needs a refined pattern (e.g., exclude lines containing "不说" / "不以" / "不主张" / "而非"). Pro: more natural academic Chinese; explicit pedagogical role. Con: G4 grep becomes context-sensitive; risk of weakening the firewall.

Cycle 039 chose Path A consistent with cycle 036 precedent. If cycle 040 surfaces > 5 negation-context corrective hits, propose Path B for cycle 041 通稿审查 discussion.
