## Key architectural decisions in S2

```text
1. Substrate hypothesis: hybrid (transformer perception + SSM/Mamba
   executive memory + slot memory for Permanence + parameter-free
   Composer + cross-spec memory bus). Evidence label: inferred.

2. Control-graph-as-architecture: the novelty is the bus + gates +
   cross-module signal protocol, not any single module's internals.

3. 4 finalist specs (Critic / Memory / Permanence / Composer)
   synthesized as 4 computational cores (C1-C5 + C6 bus) on the bus.

4. v2.1 cross-spec signal contract rendered as runtime tensor schema
   with publish/read/handoff surfaces.

5. CR-1..CR-6 conflict resolution rules rendered as architectural
   gates (hard masks, input weightings, invariants, ledgers).

6. A1-A8 actions mapped to concrete layers; A7/A8 reserved as
   bus hooks for future Cross-Modal / Active Perception specs.
```

## Key ablation plan decisions in S3

```text
1. Tier structure: 3 tiers (must-run / should-run / nice-to-have)
   with 10 ablations total (ABL-1..ABL-10).

2. ABL-1 (bus removal) is the HIGHEST priority ablation; if it fails,
   the architecture story collapses.

3. ABL-2 (substrate hypothesis) is independent of ABL-1; can run in
   parallel.

4. Per-module removal ablations (ABL-4..ABL-7) require ABL-1 to pass
   first.

5. Benchmark categories B1-B6 defined (static pair, many-view, long
   dynamic, mixed-regime, hard-case, adversarial CR-triggering).

6. Q1 (substrate ablation priority) resolved: hybrid first, SSM-only
   second, transformer-only third.

7. Q2 (adversarial vs natural CR-rule triggering) resolved: both
   (B1-B5 for performance, B6 for CR-rule verification).
```
