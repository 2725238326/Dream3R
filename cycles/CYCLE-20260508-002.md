## Code read

CUT3R:

- `/hdd3/kykt26/code/cut3r/src/dust3r/model.py`
- `/hdd3/kykt26/code/cut3r/src/dust3r/blocks.py`
- `/hdd3/kykt26/code/cut3r/src/dust3r/inference.py`

Key observed mechanisms:

- Learned `register_tokens` define fixed-size state tokens.
- State tokens get synthetic positional encoding and are projected to decoder dimension.
- `_decoder` runs paired state and image decoder streams, with state attending to frame tokens and frame tokens attending back to state.
- State persistence uses update masks and reset masks.
- `LocalMemory` is a learned cross-attention read/write table used for pose retrieval.

Spann3R:

- `/hdd3/kykt26/code/spann3r/spann3r/model.py`

Key observed mechanisms:

- `SpatialMemory` stores key tokens, value tokens, optional confidence, attention accounting, counts, points, and images.
- Memory read is normalized dot-product attention from query tokens to stored keys, followed by value aggregation and residual fusion.
- Writes can be skipped by similarity against recent working memory.
- Pruning uses accumulated attention divided by count, while protecting young entries.
- Memory read feeds into DUSt3R decoder before pointmap prediction.

Dream3R current C2:

- `/hdd3/kykt26/code/dream3r/dream3r/modules.py`
- `/hdd3/kykt26/code/dream3r/dream3r/memory_anchor_bank.py`
- `/hdd3/kykt26/code/dream3r/dream3r/nsa_attention.py`

Key observed gap:

- Current C2 centers on a single vector GRU state plus vector-only AnchorBank and NSA-style branch averaging.
- It does not yet store spatial values or point payloads.
- Memory does not yet feed a reconstruction decoder before pointmap prediction.
