## 三、写作目标

请把 PPT 内容改成一个自然、正式、清楚的开题报告叙事。建议主线如下，但可以根据材料微调：

1. `研究背景`
   - 前馈式三维重建的发展背景。
   - 传统几何流程与 3R 方法的关系。

2. `国内外研究现状`
   - DUSt3R / MASt3R / Fast3R / VGGT 等前馈式几何模型。
   - MonST3R 等动态场景方法。
   - CUT3R / Spann3R 等长序列和记忆方法。
   - Test3R 等测试时校验方法。

3. `问题提出`
   - 现有方法在长序列、动态干扰、几何一致性、多模型比较和工程复现方面仍有问题。
   - 这些问题如何自然引出本课题。

4. `研究目标与研究内容`
   - 目标不能写成已经取得最终结果。
   - 要明确本课题拟研究什么、拟实现什么、拟验证什么。

5. `候选架构设计`
   - 说明总体思路：以显式空间记忆、几何校验、专家编排和统一信号合同构成候选架构。
   - 分模块讲清楚 Perceiver、SpatialMemory / C2 Memory、Permanence、Critic、Composer 的作用。
   - 每个模块都要说明输入、输出、它解决的具体问题和后续验证方式。

6. `软件平台设计`
   - 平台是研究支撑系统，服务于多模型执行、状态记录、结果对比和实验复现。
   - 必须讲清楚它和候选架构如何配合：例如记录 Memory/Critic/Composer 输出、支持消融、管理多模型运行结果。
   - 不要编造真实 UI；平台 UI 截图目前待用户补充。

7. `实验计划与可行性`
   - 架构实验和平台实验两条证据链。
   - 已有基础、待验证内容、风险和计划。

8. `预期成果`
   - 候选架构方案。
   - 平台原型。
   - 消融和对比实验结果。
   - 开题后论文材料积累。

## 四、硬性语言要求

请严格执行：

1. 禁止使用以下句式：
   - `不是……而是……`
   - `不只是……而是……`
   - `并不是……而是……`
   - `不仅……还……` 也尽量少用。

2. 禁止 AI 味表达：
   - `赋能`
   - `闭环`
   - `抓手`
   - `底座`
   - `一体化`
   - `生态`
   - `范式升级`
   - `能力矩阵`
   - `智能化`
   - `全链路`
   - `可视化呈现`
   - `深度融合`
   - `打造`
   - `构建了完整体系`
   除非原文或技术名必须使用，否则不要写。

3. 页面文字必须像正式展示文本：
   - 不要写“这里需要说明”“可以看到”“讲解时说”。
   - 不要写口播提示。
   - 不要写长段论文文字。
   - 不要把计划写成已经完成。

4. 每页必须有一个清楚的“本页结论”：
   - 本页到底想让老师记住什么。
   - 这个结论要放在 `slide_claim` 字段中。

5. 内容必须回答下面六个问题：
   - 本课题研究什么？
   - 为什么这个问题值得做？
   - 现有方法缺在哪里？
   - 拟采用什么技术路线？
   - 软件平台承担什么作用？
   - 后续如何验证？

## 五、开题报告文风示例

推荐表述：

- `本课题关注前馈式三维重建在长序列场景下的稳定性问题。`

- `本研究拟设计一种包含显式空间记忆、几何校验和专家编排的候选架构，并通过消融实验分析各模块作用。`
- `软件平台用于统一管理多模型运行过程，为后续对比实验和结果复核提供支撑。`
- `当前工作已完成部分模型接入和流程验证，后续重点是补充真实数据实验和模块消融。`

避免表述：

- `本研究不是简单拼接模型，而是构建统一架构。`
- `平台不是展示 UI，而是证据链工作台。`
- `系统通过全链路闭环赋能研究流程。`
- `本课题实现了范式级创新。`
- `该架构显著优于现有方法。`
