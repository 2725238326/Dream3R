## Why This Card Exists (Compared To MEMORY-01)

The headline mechanism argument lives in CASE-20260504-MEMORY-01 (MonST3R 48-frame; full A1/A2/A3 differentiation). This card is the **cross-comparator argument**: Spann3R already has internal memory built into its streaming pipeline. Does Dream's GEM-3R Memory add value on top of a model that already does memory?

The answer this card argues for (paper-derived, inferred):

```text
Spann3R's internal memory operates at the latent-feature level inside
its model. Dream's GEM-3R Memory operates at the action-policy level
above any 3R model's output. The two are not in conflict; they are at
different abstraction levels. GEM-3R adds:
  - explicit anchor_set(t) protected across windows (Spann3R's internal
    memory does not surface a protected-anchor concept to consumers)
  - explicit A2 merge / ignore decisions on candidate writes (Spann3R
    does not externalize write decisions)
  - explicit policy_log(t) trace (Spann3R's memory operations are
    opaque to a consumer)
The argument: GEM-3R provides governance over an existing memory-
equipped 3R model, not a replacement for it.
```

This is a paper-grade argument, not a measurement. The card succeeds if the comparison frame is coherent (acceptance) and fails if Dream's predicted A1/A2/A3 actions duplicate what Spann3R already does internally (which would mean Dream is redundant on this regime).
