# Dream3R Demo Brief: Mamba Path Integration

Status: runnable dependency-free W17 path.

## What changed

Dream3R now has a real `state_recurrence_type="mamba_hybrid"` path. It is not just a planning flag:

- `StateTokenRecurrence`: existing CUT3R-style cross-attention state update.
- `MambaHybridRecurrence`: `mamba_ssm.Mamba(use_fast_path=False)` state evolution when available, with PyTorch selective-scan fallback, plus Transformer frame cross-attention.
- `build_state_recurrence()`: factory that switches between both paths.
- `SpatialMemory`: records `state_recurrence_type` in `memory_retrieval_log`.

This gives a concrete Mamba-style streaming state path without installing `mamba_ssm`.

## Why this is useful

The current implementation combines the previous Cycle 033 pieces into one runnable story:

- Frozen/fallback visual encoder path feeds trainable evidence heads.
- Active state tokens evolve across windows.
- Stable AnchorBank stores and recalls persistent anchors.
- NSA fuses compressed state, selected anchors, and sliding recent context.
- Critic produces geometric conflict and repair signals.
- Permanence keeps per-slot object state and ISA-style reference poses.
- Mamba-hybrid recurrence can now replace the cross-attention recurrence for streaming state evolution.

## Demo command

Run on server:

```bash
cd /hdd3/kykt26/code/dream3r
conda run -n dream3r python -m dream3r.demo_mamba_path
```

Expected output is JSON with two variants:

- `cross_attention`
- `mamba_hybrid`

Key fields to show:

- `latent_state_tokens`: state shape stays `[1, 32, 128]`.
- `state_delta_mean_abs`: state changes across windows, so recurrence is active.
- `stable_promotion_rate`: active state promotes into stable memory. The demo sets `active_to_stable_threshold=0.0` to make this path visible on random synthetic inputs.
- `nsa_branch_mean`: NSA still mixes compressed/selected/sliding branches.
- `backend`: Mamba path reports `mamba_ssm` when the server package is available, otherwise `pytorch_selective_scan`.

## Current boundary

The server currently has `mamba_ssm` available. The default fast CUDA path is not used because its `causal_conv1d` ABI can mismatch the installed extension; `use_fast_path=False` runs successfully. The PyTorch selective-scan fallback remains in place so the model still imports and tests on machines without the package.
